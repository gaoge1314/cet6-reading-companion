# CET-6 传统阅读 HTML 阅读器设计文档

## 动机

原六级阅读陪练 Skill 完全基于对话框模式，存在以下痛点：

1. **阅读体验差**：长文章在聊天框里拥挤，无法舒适浏览
2. **生词交互繁琐**：AI 需要逐个猜测用户不认识的单词，用弹窗询问效率低
3. **流程不自由**：用户无法自主切换阶段，必须跟随 AI 的节奏

## 核心思路：方案 A+C 结合

| 层面 | 方案 | 说明 |
|:----|:----|:------|
| **A: 静态 HTML** | 每个练习对应一个独立 HTML 文件 | 双击即开，零依赖，文章在浏览器中舒适阅读 |
| **C: 聊天框内独白** | AI 仍用"考生内心独白"推进对话 | 但 AI 不再展示完整文章，而是推动用户切换 HTML 页面 |

AI 的角色变为：
- 在聊天框进行考生内心独白
- 指导用户切换到相应 Phase 的页面
- 在聊天框补充生词释义、分析错题原因
- 所有文章内容、选项、阅读材料都在 HTML 页面中展示

## 生词交互：方案 B — 划词弹窗

用户**选中任意单词/短语（1-4词）** → 弹出"加入生词本"按钮 → 点击确认加入。

相比之前 AI 逐个弹窗询问的方式：
- 用户主动选择，而非被动回答
- 所有单词都可选，不限于 AI 预判的生词
- 存储于浏览器 localStorage，刷新不丢失
- 可一键导出为 Markdown 生词本文件

## 页面架构

### 全局布局

```
┌──────────────────────────────────────────────┐
│ Header: 标题 + 试卷信息 + 生词计数 + 导出按钮 │
├──────────────────────────────────────────────┤
│ Tabs: Phase 1 | 2a | 2b | 2c | 3 | 4        │
├──────────────────────┬───────────────────────┤
│                      │  📝 生词本面板         │
│   主内容区            │  (实时更新)            │
│   (当前 Phase 内容)   ├───────────────────────┤
│                      │  📋 答题卡             │
│                      │  (Phase 3 显示答案)     │
└──────────────────────┴───────────────────────┘
```

### 6 个 Phase 内容

| Phase | 内容 | 关键功能 |
|:-----:|:-----|:---------|
| **1** 扫描 | 主题预测、题型分类、5题完整题干+选项 | 划词加生词 |
| **2a** 首尾定中心 | 10段首句+尾句、AI内心独白、中心归纳 | 划词加生词 |
| **2b** 中心做题 | 可用中心做的题(46,50)标记暂选，其余标待精读 | 划词加生词 |
| **2c** 定位精读 | 定位栏展示映射关系，精读3段+做3题 | 划词加生词 |
| **3** 对答案 | 正确率、逐题分析、技巧归属 | 答题卡自动更新 |
| **4** 复盘 | 技巧统计、生词汇总、薄弱环节 | 导出所有生词 |

## 关键技术决策

### 数据持久化
- **生词本**：localStorage，key = `cet6-passage1-vocab`
- 导出格式：Markdown 表格，可直接保存为 `.md` 文件
- 无需后端，无需数据库

### 正确答案来源
- 从 `2025.12六级解析第1套.pdf` 提取官方答案
- 验证结果：46-D, 47-A, 48-D, 49-B, 50-B（全对 5/5）

### 交互细节
- 划词弹窗仅限主内容区 (`.content`)，不影响侧边栏操作
- 弹窗跟随鼠标位置，不超过窗口边界
- 选中后自动清除选区，避免干扰阅读
- 滚动内容时自动关闭弹窗

### 样式
- 跟随系统暗色模式 (`prefers-color-scheme: dark`)
- CSS 变量主题系统
- 响应式布局，手机端自动堆叠为单列

## 使用流程

1. AI 在聊天框用考生内心独白开始做题
2. 用户打开对应的 HTML 文件（双击）
3. AI 指导： "先看 Phase 1 扫描题干和选项"
4. 用户在页面中点 Tab 切换到 Phase 1，阅读并划词记录生词
5. AI 继续独白 → "现在切换到 Phase 2a 看首尾句"
6. 用户切换到 Phase 2a，阅读首尾句，划词
7. 循环至 Phase 4 完成
8. 用户点击"导出 .md"保存生词本

## 与原有 Skill 的关系

- **SKILL.md 不变**：核心流程（Phase 1→2a→2b→2c→3→4）和考生内心独白语气保持不变
- **新增工具**：HTML 阅读器作为视觉辅助工具
- **AI 角色调整**：不再在聊天框展示完整文章，转而推动用户使用 HTML 页面
