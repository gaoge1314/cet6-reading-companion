# 六级长篇阅读（匹配题）辅助学习程序 · 设计方案

## 概述

基于刘晓艳老师"四六级最后30天冲刺"讲座中的长篇阅读做题建议，为六级长篇阅读（匹配题）设计辅助学习程序。该程序复用现有 `cet6-reading-platform` 的架构和组件，针对匹配题的特性进行适配。

## 老师核心建议

### 做题策略
- **时间**：严格15分钟
- **目标**：做对7-8道即为全国王者
- **心态**：出题人本意考察单位时间内的定位能力，并非要求全做完
- **放弃原则**：一道题超过1分钟找不到，立即放弃

### 三层定位法
| 优先级 | 定位依据 | 说明 |
|-------|---------|------|
| ① 最佳 | 专有名词（时间、地点、人名、地名、数字） | 1-2道一定有 |
| ② 备用 | 普通名词 + 动词 | 注意同义替换 |
| ③ 兜底 | 形容词/副词比较级和最高级 | 形容词易替换，比较级/最高级是锚点 |

## 程序架构

### 5-Phase流程

| Phase | 名称 | 功能 |
|-------|------|------|
| Phase 1 | 选项扫读 | 浏览10个选项，识别定位词类型 |
| Phase 2 | 计时定位匹配 | 15分钟限时，左右分栏匹配 |
| Phase 3 | 对答案 | 核对答案 + 耗时统计 |
| Phase 4 | 深度复盘 | 耗时分布图、放弃策略评估、同义替换展示 |

### 核心功能

1. **15分钟倒计时器**：5分钟/10分钟/1分钟分段提醒
2. **单题计时器**：每题独立计时，超过60秒弹出放弃建议
3. **段落匹配交互**：选项右侧显示段落按钮(A-H)，点击即可匹配
4. **放弃/恢复机制**：每题独立放弃按钮
5. **耗时分布图**：Phase 4展示每题耗时柱状图（绿≤60s、黄60-90s、红>90s）
6. **放弃策略评估**：分析放弃的题是否合理
7. **同义替换展示**：原文与选项的关系对照

### 复用组件

- `cet6-loader.js`：数据加载
- `cet6-vocab.js`：生词本
- `cet6-annotate.js`：划线标注
- `theme.css` / `layout.css`：主题和布局样式
